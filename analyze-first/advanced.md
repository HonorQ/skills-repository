## 定制化选项

### 用户偏好设置

用户可以通过以下方式定制分析行为：

#### 1. 方案类型偏好

```
"只给我简单方案" / "only simple solutions"
→ 只推荐低复杂度方案，忽略复杂方案

"给我所有方案，包括复杂的"
→ 包含所有可行方案，无论复杂度
```

#### 2. 约束条件

```
"不要引入新依赖" / "no new dependencies"
→ 只推荐不引入新依赖的方案

"时间紧迫，给我最快的方案"
→ 优先推荐工作量最小的方案

"这是长期项目，考虑可维护性"
→ 优先推荐长期可维护的方案
```

#### 3. 分析深度

```
"快速分析一下" / "quick analysis"
→ 快速深度：2-3 个方案，简洁对比

"详细分析" / "deep dive"
→ 深度分析：4-5 个方案，完整评估
```

#### 4. 输出格式

```
"用表格对比" / "table format"
→ 使用表格展示方案对比

"给我详细报告"
→ 完整的分析报告

"简洁一点"
→ 精简输出，只保留关键信息
```

#### 5. 风险偏好

```
"避免高风险方案" / "avoid high-risk approaches"
→ 只推荐低风险方案

"可以接受中等风险"
→ 包含中等风险方案

"这是实验性项目，可以冒险"
→ 包含高风险高回报方案
```

### 组合使用

可以组合多个偏好：

```
"快速分析，不要新依赖，避免高风险"
→ 快速深度 + 无新依赖 + 低风险

"详细分析，考虑长期可维护性，先做 POC"
→ 深度分析 + 长期优先 + POC 验证
```

## 分析深度控制

### 自动深度判断

AI 根据问题特征自动判断深度：

| 特征 | 深度 | 原因 |
|------|------|------|
| 简单工具选择 | 快速 | 选项明确，影响小 |
| 功能实现 | 标准 | 需要权衡多种因素 |
| 架构决策 | 深度 | 影响长期，风险高 |
| 性能优化 | 标准/深度 | 取决于优化范围 |
| 重构 | 深度 | 影响广泛，需要谨慎 |

### 手动深度控制

用户可以手动指定深度：

```
"快速分析一下" → 快速深度
"详细分析" → 深度分析
"deep dive" → 深度分析
"just give me a quick answer" → 快速深度
```

### 深度调整建议

AI 可以建议调整深度：

```
AI: "这个问题涉及架构决策，建议使用深度分析。是否需要详细分析？"
用户: "是的，详细分析一下"
AI: [切换到深度分析]
```

## 特殊情况处理

### 方案都不满意

如果用户对所有方案都不满意：

```
用户: "这些方案都不太合适"
AI: "可以尝试以下方式：
1. 调整约束条件（如时间、资源）
2. 重新定义问题范围
3. 寻找外部参考

是否需要重新分析？"
```

### 分析结果与预期不符

如果分析结果与用户预期不符：

```
用户: "我没想到会推荐这个方案"
AI: "让我解释一下推荐理由：
1. [理由1]
2. [理由2]
3. [理由3]

如果您的优先级不同，我可以调整推荐。您最看重哪些方面？"
```

### 需要推翻之前的决策

如果需要推翻之前的决策：

```
用户: "我觉得之前选的方案不合适"
AI: "可以重新分析。请告诉我：
1. 之前方案的哪些方面不符合预期？
2. 现在的约束条件有什么变化？
3. 有没有新的需求或限制？

我将基于新的信息重新分析。"
```

### 分析范围太大

如果分析范围太大：

```
用户: "我想重构整个应用"
AI: "这个范围太大，建议分阶段进行：
1. 先确定重构目标和优先级
2. 选择一个模块作为试点
3. 验证重构方法的有效性
4. 逐步扩展到其他模块

是否需要先分析试点模块？"
```

### 缺少关键信息

如果缺少关键信息：

```
AI: "为了给出更准确的分析，我需要了解：
1. 项目的技术栈是什么？
2. 有没有时间或资源限制？
3. 有没有团队偏好或经验限制？

请提供这些信息，我可以给出更针对性的建议。"
```
